# Reel — локальный музыкальный плеер (PWA)

Мини-приложение проигрывает **твои собственные аудиофайлы** прямо с устройства.
Ничего никуда не загружается — файлы читаются локально через браузер.

## Как запустить

Открывать `index.html` через `file://` нельзя — часть API (Service Worker,
доступ к папке) работает только по `http(s)`. Нужен любой простой сервер:

**Вариант 1 — GitHub Pages / Netlify / Vercel (бесплатно, самый простой способ)**
1. Загрузи содержимое этой папки в репозиторий или перетащи в Netlify Drop.
2. Открой полученную ссылку на телефоне.
3. В Chrome (Android) нажми «Установить приложение» / «Добавить на главный экран».

**Вариант 2 — локально для теста (с компьютера)**
```bash
cd music-player
python3 -m http.server 8080
```
Открой `http://localhost:8080` (или IP компьютера в локальной сети — с телефона).

## Как пользоваться
1. Нажми «Добавить музыку» и выбери папку с аудиофайлами (mp3, m4a, wav, ogg, flac).
2. Плеер найдёт все аудиофайлы в папке (включая вложенные подпапки) и покажет список.
3. Играй как обычно: свайпай экран блокировки, используй наушники — управление
   работает через системный интерфейс (Media Session API).

## Честно об ограничениях платформ

- **Android, Chrome, установлено на главный экран** — фоновое воспроизведение
  и управление с экрана блокировки работают полноценно, как в обычном плеере.
- **iOS, Safari** — веб-стандарт `File System Access API` (постоянный доступ
  к папке) там **не поддерживается**, поэтому при каждом запуске придётся
  заново выбирать файлы (через системный выбор файлов — это ограничение
  Apple, не этого приложения). Фоновое воспроизведение при заблокированном
  экране на iOS тоже менее стабильно, чем в нативных приложениях или в PWA
  на Android — Apple специально ограничивает это для не-App Store приложений.
- **Desktop Chrome/Edge** — всё работает как на Android.

Если фоновая работа на iOS критична, единственный по-настоящему надёжный
путь — нативное приложение через Xcode (AVAudioSession + `MPNowPlayingInfoCenter`),
это отдельная, более крупная задача — могу помочь, если понадобится.

## Структура файлов
```
index.html    — разметка
styles.css    — стили (тема катушечного магнитофона)
app.js        — вся логика: доступ к файлам, транспорт, Media Session
sw.js         — офлайн-кэш оболочки приложения (не аудио)
manifest.json — установка как PWA
icons/        — иконки для домашнего экрана
```
